## 4. 功能展示设计

报告使用本地实际运行系统的截图，不使用设计稿或模拟图片。至少展示以下四项：

1. 登录界面：说明手机号登录/注册和身份认证入口。
2. 智能对话：展示一次校园制度问题、流式回答、来源引用及会话记录。
3. 知识库资料：展示系统挂载的学生手册、奖学金办法和综合素质测评办法等资料。
4. PDF 原文预览：展示引用资料的真实 PDF 页面、页码和翻页能力。

每张图片设置编号和图注，并在正文中解释该功能解决的问题，而不是只堆放截图。

实验部分以仓库中的测试用例、实验脚本和可复核结果为依据，对比 baseline、关键词重排、查询改写、BM25 与向量混合检索等策略。正式报告前重新核对指标，解决 README 中 66.7% 与现有测试报告 90% 的口径差异；若二者来自不同测试集，则分别注明测试集规模、评价方法和实验日期，不将它们直接比较。

报告中的功能、技术栈、接口和数据规模均以代码和实际运行结果为准。无法复核的数字不写成确定结论。

## 6. 排版规范

- 页面：A4 纵向，适合 Word 打印与电子提交。
- 中文：宋体。
- 英文、数字和英文标点：Times New Roman。
- 标题：继续使用宋体，通过字号和加粗建立层级，不混用黑体。
- 正文：小四号，1.5 倍行距，首行缩进 2 字符。
- 图表：统一编号，图片居中，图题置于图片下方。
- 页眉页脚：保持简洁，页脚使用连续页码。
- 参考文献：采用统一的中文课程报告引用格式。只引用能够在互联网公开检索的论文、标准或官方技术文档，优先使用论文出版页、DOI、会议官网和项目官方文档，不使用无法核验来源的博客或聚合转载页。
- 链接：每条网络参考文献均附可直接访问的完整链接，并在 Word 中设置为可点击的超链接；交付前逐条检查标题、作者、年份和链接是否对应。
