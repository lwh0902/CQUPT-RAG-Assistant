# CQUPT-RAG 报告事实清单

## 项目定位

- 项目是面向重庆邮电大学校园制度查询的 Web 智能问答系统。
- 前沿技术定位为大语言模型应用、RAG、语义向量检索、混合信息检索和 Function Calling，而非自行训练基础大模型。
- 前端为 React 19 + TypeScript，后端为 FastAPI，实时通信使用 WebSocket。

## 知识库

- `manual_index_meta.json` 记录的最近一次建库时间为 2026-06-22 10:11:39。
- 共挂载 4 份制度文档、201 页、329 个文本块。
- 文档包括学生手册、校级奖学金评定办法、综合素质测评办法和社会奖学金评定办法。
- PDF 与 DOCX 经加载后使用递归文本切分，代码中的块大小为 600 字符、重叠 80 字符。
- 向量数据库为持久化 Chroma 集合 `cqupt_policy_docs`。
- 向量模型由 `vector_store.py` 明确配置为智谱 `embedding-2`，并以每批 16 条文本调用接口。

## 检索与生成

- baseline：纯向量检索。
- rerank：向量召回后按校园领域关键词重排。
- rewrite：GLM-4.7-Flash 将一个问题改写成 2—4 个子查询，分别向量召回后去重并重排。
- hybrid：每个子查询同时进行向量检索与 BM25 检索，再以 RRF 公式融合排名。
- 系统先取 8 条候选，最终选择 3 条上下文供模型作答。
- 提示词要求只能根据检索资料回答；资料中无明确答案时返回“无法确定”。
- 生成模型由代码配置为 `glm-4.7-flash`，禁用思考输出，并通过 WebSocket 流式返回文本。

## Agent 与记忆

- Function Calling 工具包括天气查询和课表查询；当前均为可演示的本地模拟数据工具，不宣称连接真实教务系统或实时天气服务。
- 第一轮模型决定是否调用工具，程序执行函数后把 `tool` 消息回填，第二轮生成面向用户的回答。
- 短期记忆读取 MySQL 会话消息；长期记忆将历史窗口摘要写入独立 Chroma 集合，并在后续对话中按语义检索。

## 可展示功能

- 手机号注册/登录与 JWT 鉴权。
- 新建会话、历史会话列表、全文搜索与会话重命名。
- WebSocket 流式问答和思考步骤提示。
- 回答来源列表、原文片段、PDF 页码缩略图和原页预览。
- 知识库资料列表展示文档名称、类型、大小和页数。

## 实验口径

- 正式实验文件为 `experiment_report.json`，共 12 条结构化测试用例。
- 题型分布：事实类 5 条、规则类 4 条、多条件类 2 条、拒答类 1 条。
- baseline：5/12，41.7%，平均 8.16 秒。
- rerank：7/12，58.3%，平均 12.23 秒。
- rewrite：8/12，66.7%，平均 25.47 秒。
- hybrid：5/12，41.7%，平均 31.01 秒。
- 在本次小规模测试中，rewrite 准确率最高；hybrid 没有取得更好结果，说明关键词分词、语料规模、融合参数和检索链路仍需调优。
- `test_report.md` 仅写有 18/20（90%）且省略全部用例明细，无法与 12 条实验直接比较，因此不作为正式结论。
